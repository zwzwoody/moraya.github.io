# SEO Title Optimizer

帮助用户优化博客文章标题，提升搜索引擎排名和点击率。

## 快速开始

```
帮我优化标题 SEO
原始标题: 如何学习编程
```

## 说明

### 第零步：读取博客信息（自动执行）

在开始优化标题前，先读取用户的博客信息：

1. **读取博客配置**
   - 读取 `_config.yml` 获取博客标题、描述、作者
   - 读取主题配置文件了解博客风格

2. **读取现有文章**
   - 扫描 `source/_posts/` 目录下的文章
   - 了解博客的内容主题、风格、已有的标题模式
   - 分析文章的 front matter（tags、categories）

3. **分析博客定位**
   - 了解博客的主要读者群体
   - 了解博客的内容风格（技术/生活/专业）
   - 确保生成的标题与博客整体风格一致

### 第一步：获取原始标题

提示用户输入需要优化的原始标题，支持：
- 单独输入标题
- 提供标题 + 文章关键词
- 提供标题 + 文章摘要/内容概述

### 第二步：分析 SEO 因素

从以下维度分析标题：

1. **长度分析**
   - 最佳长度：50-60 字符（Google 显示上限）
   - 移动端：30-40 字符
   - 过短：信息不足
   - 过长：被截断

2. **关键词优化**
   - 主关键词位置（前部优先）
   - 关键词密度
   - 关键词相关性
   - 长尾关键词建议

3. **吸引力提升**
   - 情感词使用
   - 数字效应
   - 问句形式
   - 悬念制造

4. **结构建议**
   - 分隔符选择（|、-、:）
   - 品牌词位置
   - 避免重复词

### 第三步：生成优化方案

生成 3-5 个优化后的标题选项，每个包含：

- **优化后的标题**
- **简要说明**（为何这个标题更好）
- **适用场景**

### 第四步：用户选择

让用户选择：
- 采用某个优化后的标题
- 进一步调整要求
- 继续优化其他标题

## SEO 优化规则

### 标题长度

| 类型 | 字符数 | 说明 |
|------|--------|------|
| 最佳 | 50-60 | Google 完整显示 |
| 移动端 | 30-40 | 移动搜索显示 |
| 最小 | 20 | 信息充足 |
| 最大 | 70 | 可能被截断 |

### 关键词位置

| 位置 | 权重 |
|------|------|
| 前 10 字符 | 最高 |
| 前 20 字符 | 高 |
| 中间 | 中 |
| 结尾 | 低 |

### 高效词汇

**情感词**：
- 免费、限时、终极、完整
- 秘密、揭秘、必备、必看
- 快速、高效、简单、轻松

**数字词**：
- 10 个方法、5 分钟、3 步
- 2026、最新、终极版

**问题词**：
- 为什么、如何、怎样
- 是什么、哪个、多少

### 避免事项

- 避免标题党（夸大其词）
- 避免重复词
- 避免特殊字符（除分隔符外）
- 避免全部大写

## 最佳实践

1. **结合博客定位**，确保标题与博客整体风格一致
2. **参考现有文章风格**，使用类似的命名模式
3. **主关键词放前面**，权重最高
4. **控制长度在 60 字符内**，避免截断
5. **使用数字**，增加可信度和点击欲
6. **添加情感词**，提升吸引力
7. **提供明确价值**，让用户知道能得到什么
8. **测试不同版本**，A/B 测试优化

## 故障排除

**标题过于笼统**：
- 添加具体限定词
- 加入数字量化
- 明确目标受众

**关键词堆砌**：
- 控制在 2-3 个核心词
- 使用同义词扩展
- 自然流畅表达

**没有吸引力**：
- 加入情感触发词
- 使用问题形式
- 制造紧迫感或好奇心
